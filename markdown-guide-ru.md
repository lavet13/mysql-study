# Руководство по синтаксису Markdown → DOCX

В этом файле описано, как синтаксис Markdown в `report.md` превращается
в форматирование Word после конвертации через Pandoc с использованием
`templates/reference.docx`.

---

## Заголовки → стили Word Heading

```markdown
# 1. Настройка среды          → Heading 1 (жирный, без отступа, отбивка сверху)
## 1.1 Подраздел              → Heading 2 (жирный, без отступа)
### 1.1.1 Подподраздел        → Heading 3 (жирный курсив, без отступа)
```

**Правило:** Нумерацию заголовков всегда пишите вручную (`1.`, `1.1`).
Pandoc не нумерует автоматически.

---

## Основной текст → стиль Normal

Любой обычный абзац получает стиль `Normal`:
Times New Roman 14pt, межстрочный интервал 1,5, отступ первой строки 1,25 см,
выравнивание по ширине.

```markdown
Это обычный абзац. Он будет отформатирован с отступом первой строки
и выравниванием по ширине.
```

**Важно:** Между абзацами обязательно оставляйте **пустую строку**.
Без неё Pandoc считает две строки одним абзацем.

```markdown
Первый абзац.

Второй абзац.   ← пустая строка обязательна
```

---

## Жирный и курсив

```markdown
**жирный текст**        → жирный
*курсив*                → курсив
***жирный курсив***     → жирный курсив
```

---

## Блоки кода → стиль Source Code

Оборачивайте код в тройные обратные кавычки с указанием языка:

````markdown
```sql
SELECT * FROM Employees WHERE dept_id = 128;
```

```yaml
services:
  db:
    image: mysql:8.0
```

```bash
docker compose up -d
```
````

Результат: Courier New 10pt, одинарный интервал, отступ блока слева.

**Важно:** Тег языка (`sql`, `yaml`, `bash`) необязателен, но это хорошая
практика. В нашем шаблоне все блоки кода используют стиль `Source Code`
вне зависимости от тега.

## Таблицы → стиль Table (с рамками, Times New Roman 14pt)

```markdown
| Код сотр. | ФИО         | Должность   |
|-----------|-------------|-------------|
| 7513      | Иванов И.И. | Программист |
| 9842      | Сергеева С. | Админ БД    |
```

**Правила:**
- Строка-разделитель `|---|---|---|` **обязательна** — без неё Pandoc не распознаёт таблицу
- Ширина столбцов подбирается автоматически по содержимому
- Первая строка (над разделителем) — заголовок таблицы
- Текст ячеек использует стиль `Compact`: TNR 14pt, интервал 1,15, без отбивки

**Выравнивание в столбцах** (необязательно):

```markdown
| Левый     | По центру   | Правый      |
|:----------|:-----------:|------------:|
| текст     | текст       | текст       |
```

## Изображения → встраиваются в документ

```markdown
![Рисунок – Таблица сотрудников](assets/employees.png)
```

- Путь указывается **относительно `report.md`** — то есть `assets/` означает `lab-01/assets/`
- Текст в `[]` — это подпись (показывается если изображение не найдено)
- Скриншоты кладите в `lab-XX/assets/` до запуска сборки

**Если изображение отсутствует**, Pandoc выводит предупреждение и вставляет
вместо него текст подписи в скобках — сборка при этом не прерывается.

---

## Горизонтальная линия (только визуальный разделитель)

```markdown
---
```

Вставляет горизонтальную линию. Удобно для визуального разделения разделов
в исходном файле. **Не является** разрывом страницы.

---

## Принудительный разрыв страницы

В Markdown нет специального синтаксиса для разрыва страницы.
При необходимости используйте raw-div:

```markdown
::: {custom-style="pagebreak"}
:::
```

На практике заголовки `Heading 1` сами по себе хорошо отбивают содержимое,
и ручные разрывы страниц в теле отчёта почти никогда не нужны.

## Списки

```markdown
- Первый элемент        ← маркированный список
- Второй элемент
  - Вложенный           ← вложение через 2 пробела

1. Первый шаг           ← нумерованный список
2. Второй шаг
```

**Важно:** Перед первым элементом списка оставляйте пустую строку,
если перед ним идёт абзац — иначе Pandoc может не распознать список.

## Встроенный код (одиночные обратные кавычки)

```markdown
Используйте команду `docker compose up -d` для запуска.
```

Отображается моноширинным шрифтом внутри обычного абзаца.

---

## Что НЕ включать в report.md

- **Титульный лист** — формируется отдельно через `templates/merge.py`
- **YAML front matter** (блок `---` в самом начале файла) — не нужен
- **HTML-теги** — не рендерятся корректно в docx
- **LaTeX** — по той же причине
