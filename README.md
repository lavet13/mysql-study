# Лабораторная работа: Нормализация БД

Данный проект демонстрирует процесс нормализации базы данных от ненормализованного вида к **3-й нормальной форме (3НФ)** на примере системы учета квалификации сотрудников и отделов. Проект развернут с использованием Docker, MySQL и Prisma ORM.

## 🛠 1. Подготовка окружения

Для запуска проекта локально устанавливать базы данных или Node.js не требуется. Нужен только **Docker Desktop** (он включает Docker Engine и Docker Compose).

### Ссылки на скачивание Docker:
* [Windows](https://www.docker.com/products/docker-desktop/) *(рекомендуется использовать с WSL2)*
* [macOS](https://www.docker.com/products/docker-desktop/) *(Intel или Apple Silicon)*
* [Linux](https://docs.docker.com/desktop/install/linux-install/)

---

## 🚀 2. Инструкция по запуску

### Клонирование проекта
Склонируйте репозиторий на свой компьютер и перейдите в папку проекта:
```bash
git clone <ссылка_на_ваш_репозиторий>
cd <название_папки_проекта>
```

(Если вы скачали проект архивом, просто распакуйте его и откройте эту папку в терминале).

### Развертывание
Выполните команду в корневой папке проекта:
```bash
docker compose up --build -d
```

* Флаг `--build` принудительно собирает актуальные образы.
* Флаг `-d` запускает контейнеры в фоновом режиме.

**Важно:** Подождите 15-20 секунд. За это время MySQL инициализирует базу данных из файла `init.sql`, а Prisma подготовит интерфейс.

## 📊 3. Работа с данными (Prisma Studio)
После успешного запуска контейнеров откройте браузер и перейдите по адресу:
👉 **http://localhost:5555**

Здесь откроется панель управления Prisma Studio. Вы сможете визуально просматривать созданные таблицы (**Employees**, **Departments**, **EmployeeQualifications**), а также добавлять, редактировать или удалять записи.

## 🛠 4. Полезные команды
Все команды выполняются в терминале из корневой папки проекта.

### Пересоздание базы "с чистого листа"
```bash
docker compose down -v
docker compose up -d
```

### Обновление структуры через Prisma
Если вы изменили файл prisma/schema.prisma и хотите применить новую структуру к базе данных:
```bash
docker compose exec prisma npx prisma db push
```

### Синхронизация схемы из базы (pull)
Если структура базы изменилась (например, после ручных правок в init.sql или внешних миграций) и вы хотите обновить schema.prisma под текущую БД:
```bash
docker compose exec prisma npx prisma db pull
```

### Остановка проекта
```bash
docker compose stop
```

## 📖 Теоретическая справка: Нормализация в проекте
В исходном задании таблица имела нарушения правил нормализации. В ходе работы база была приведена к **3НФ**:

1. **Первая нормальная форма (1НФ — Атомарность)**: Поле Qualifications содержало списки через запятую (например, "C, Java"). Навыки вынесены в отдельную таблицу **EmployeeQualifications**.
2. **Вторая нормальная форма (2НФ)**: Каждый неключевой атрибут полностью зависит от первичного ключа.
3. **Третья нормальная форма (3НФ — Устранение транзитивности)**: Название отдела зависело от номера отдела, а не от сотрудника. Информация об отделах вынесена в таблицу **Departments**. Теперь у сотрудников только внешний ключ `dept_id`.
